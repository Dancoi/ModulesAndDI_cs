# Модульная система на C# с DI

## Описание

Данный проект реализует шаблон приложения на C# с поддержкой модульности и внедрения зависимостей (Dependency Injection). Модули автоматически обнаруживаются, их порядок и зависимости определяются через конфигурацию.

## Основные компоненты

- **Модули**: Каждый модуль реализует интерфейс `IAppModule`, может объявлять зависимости и регистрировать сервисы.
- **Каталог модулей**: Класс `ModuleCatalog` отвечает за обнаружение, проверку зависимостей и определение порядка инициализации модулей.
- **Действия модулей**: Модули могут добавлять действия (`IAppAction`), которые выполняются после инициализации.
- **DI-контейнер**: Используется стандартный `IServiceCollection` для регистрации и разрешения зависимостей.
- **Конфигурация**: В файле `appsettings.json` задаётся список активных модулей.

## Структура

- `Core/` — интерфейсы и базовые классы для модулей и действий.
- `Modules/` — реализации модулей (например, CoreModule, LoggingModule и др.).
- `Services/` — сервисы, предоставляемые модулями (например, IStorage, InMemoryStorage, IClock).
- `Program.cs` — точка входа, отвечает за загрузку конфигурации, инициализацию и запуск модулей.

## Пример запуска

1. В `appsettings.json` укажите нужные модули.
2. Запустите проект — модули будут обнаружены, зависимости проверены, сервисы зарегистрированы, а действия выполнены.

## Пример конфигурации

```json
{
	"Modules": [
		"Core",
		"Logging",
		"Validation",
		"Export",
		"Report"
	]
}
```

## Возможности расширения

- Добавляйте новые модули, реализуя `IAppModule`.
- Определяйте зависимости между модулями через свойство `Requires`.
- Реализуйте собственные сервисы и действия.
